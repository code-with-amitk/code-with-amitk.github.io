What is DPOP Modes of Operation

DPOP (Dataplane on Premisis)?

Netskope placed a Box(Appliance) on customer premises for Packet Inspection. 

---------Customer Env------------------------
|                                            |
|   user1 -----\                             |
|               DPOP(does packet inspection)----forward to NS Cloud(if configured)--> [NS Cloud]
|   user2 -----/                             |
|--------------------------------------------
Benefits of DPoP:
1. Policies are managed in the cloud but enforced locally.
2. Local inspection avoids sending all traffic to the cloud.
3. Can be deployed as a virtual or hardware appliance.

Modes of Operation

Name nsclient needed Description
Inline (Transparent) No Traffic forwarded using (PAC file, WCCP, or routing rules)
Supports layer 4 (non-HTTP) traffic inspection.
Explicit Proxy Yes Netskope Client (NSC) or browser/OS proxy settings to send traffic to DPoP.
Best for remote users or branch offices.
Hybrid Mode - Combines inline and explicit proxy modes for flexible deployment.
